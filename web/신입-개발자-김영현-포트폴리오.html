블록 업로드 메타데이터 저장 방식 성능 검증

문제

  • 파일을 블록 단위로 업로드하는 과정에서 메타데이터를 저장할 때 지연이 발생할 가능성이 있었음
  • 1개의 블록(2MB)이 업로드 될 때마다 DB에 정보를 insert 하는 구조여서 성능 저하가 우려되었음

해결

  • 파일 블록 업로드 시나리오를 구성해 테스트 진행
  • k6 성능 테스트를 이용해 파일 블록 정보를 Redis에 저장하는 시나리오와 Mysql에 저장하는 방식 비교 분석

결과

  • 동일 조건의 블록 업로드 부하 테스트에서 Redis 저장 방식은 MySQL 대비 p95/p99 지연시간을 최대 28배까지 낮추고, 처리량은 최대 약 5배 이상 높이는 경향을 확인함
  • 특히 5GB 대용량 구간에서는 MySQL이 일부~전부 TIMEOUT을 보인 반면, Redis는 비교 구간 전반에서 더 안정적인 응답 지연을 유지했음

배치 큐 기반 비동기 처리 구조 구현

문제

  • 파일/디렉터리 삭제, 디렉터리 레벨 관리, 권한 관리 시 하위 파일과 디렉터리까지 모두 수정해야 해 처리 비용이 커지는 문제가 있었음

해결

  • 배치 테이블을 구성하고, 동일한 작업 유형의 수정 요청을 몇 초 간격으로 수집한 뒤 한 번에 처리하도록 설계

결과

  • 작업을 일괄 처리하는 방식으로 응답 속도를 개선하고 DB 부하를 줄임
